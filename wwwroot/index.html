Number of VMs
Why select a count? Batch requests create multiple identical VMs using the same resource profile and naming scheme.

Why increase CPU? More vCPU helps when many VMs are active and you run parallel services across the fleet.

Why increase RAM? Enough memory per VM prevents paging across the batch (common issue when many VMs run on one host).

Why increase storage? Multiply disk sizing by the VM count to estimate total storage impact on the host.

Pricing

VM provisioning is priced by deployment type (Single VM vs Company Batch), plus the selected resource profile (vCPU, RAM, Storage). This is an HND demonstration model for structuring service tiers (not live billing).

Single VM Provisioning

Designed for individual systems, labs, testing workloads, or a single service VM. Includes creation, resource allocation, naming, and job logging.

Base build fee: £15 per VM Includes: name + CPU/RAM/Storage allocation Output: job ID + recorded status
Resource Included Additional Cost Notes
CPU 2 vCPU +£3 per additional 2 vCPU Scaling for multi-service workloads and parallel tasks.
RAM 4 GB +£2 per additional 4 GB Reduces paging; better for IDEs, databases, and heavy apps.
Storage 40 GB +£1 per additional 20 GB More space for OS updates, apps, logs, and data growth.

Example: 4 vCPU / 8 GB / 80 GB = Base £15 + CPU £3 + RAM £2 + Storage £2 → £22

Company Batch VM Deployment

Designed for departments, labs, training environments, or small application fleets. Batch deployments use a shared profile and automated naming (e.g. PREFIX-01 … PREFIX-N).

Batch setup fee: £25 per request Includes: name generation + batch plan output Best for repeatable multi-VM builds
Profile Resources Price per VM Use Case
Standard 2 vCPU / 4 GB / 40 GB £12 Basic services, lab desktops, training VMs.
Performance 4 vCPU / 8 GB / 80 GB £18 Application servers, heavier labs, multi-user tools.
High Performance 8 vCPU / 16 GB / 120 GB £28 Databases, CI workloads, compute-heavy services.

Example: 10 Standard VMs = Setup £25 + (10 × £12 = £120) → £145

Host Resource Consideration

Batch builds multiply resource impact. Plan against host limits to avoid CPU contention, RAM pressure, and disk I/O bottlenecks.

Total CPU = Count × vCPU Total RAM = Count × RAM Total Storage = Count × Storage
